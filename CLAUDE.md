## Overview

Personal portfolio site (`m074med`) built with Flutter and primarily targeting **Flutter Web**. Deployed to Firebase Hosting (https://m074med.web.app) and GitHub Pages. CI pins Flutter `3.29.3`; the Dart SDK constraint is `>=2.12.0 <4.0.0`.

## Commands

```bash
flutter pub get                         # install dependencies
flutter run -d chrome                   # run locally in the browser (primary target)
flutter analyze                         # static analysis / lint
flutter test                            # run all tests
flutter test test/widget_test.dart      # run a single test file

# Code generation (freezed / json_serializable) — required after editing any
# class annotated with @freezed or models under lib/infrastructure/
dart run build_runner build --delete-conflicting-outputs

flutter build web --release             # production web build → build/web
```

Secrets are **not** committed. `dotenv` (the env file, see `.gitignore`) is loaded at runtime via `flutter_dotenv` with `isOptional: true`. A fresh clone won't have it; create one with the keys listed in the workflows (`EMAILJS_*`, `FIREBASE_*`) to exercise the contact form and Firestore locally. In CI the same values are also injected as `--dart-define`s, and the code prefers `String.fromEnvironment` over `dotenv.env` when present (see `lib/infrastructure/api/email_api.dart`).

## Architecture

The app is **content-driven and mostly static**. Almost every page is a stateless widget that renders hardcoded data; the only stateful/networked feature is the contact form. Layers under `lib/`:

- **`values/`** — the content + design-system layer, and where most edits happen. Files are stitched together with Dart `part of 'values.dart'`, so they share one library and are imported via `package:m074med/values/values.dart`. All site content lives here as static lists on the `Data` class in `values/data.dart` (`Data.projects`, `Data.noteworthyProjects`, `Data.certificationData`, `Data.experienceData`, `Data.socialData`, `Data.menuItems`, skills, etc.). `colors.dart`, `strings.dart`, `sizes.dart`, `styles.dart`, `images.dart`, `gradients.dart` round out the design tokens. **To add a project/certification: add an entry to the relevant list in `data.dart`, add asset paths in `images.dart`/`strings.dart`, drop files under `assets/...`, and register the asset folder in `pubspec.yaml`.**
- **`presentation/`** — UI. `pages/` (one folder/file per route, larger pages split into `widgets/` subfolders), shared `widgets/`, and `routes/routes.dart`.
- **`infrastructure/`** — the contact-form feature only: `api/` (EmailJS + Firestore), `bloc/` (`flutter_bloc`), `failures/` (`dartz` `Either` + freezed unions).
- **`core/`** — cross-cutting helpers: `layout/adaptive.dart`, `utils/`.

### Routing
Custom RegEx-based router in `presentation/routes/routes.dart`. `RouteConfiguration.onGenerateRoute` matches `settings.name` against an ordered `paths` list (first match wins, so order matters). Each page class exposes its route as a `static` string constant (e.g. `HomePage.homePageRoute`) and pages are pushed by name. Project detail uses a path parameter: `ProjectDetailPage.generateRoute(projectId)` → `/project-detail/<id>`, and the page resolves it via `Data.projects.firstWhere((p) => p.projectId == id)`. Transitions are disabled via `NoAnimationMaterialPageRoute`. Clean URLs (no `#`) come from `PathUrlStrategy` set in `configure_web.dart`.

### State management & DI
`flutter_bloc` for the email feature only. Dependencies are wired with `get_it` in `lib/injection_container.dart` (`configureDependencies()` called from `main()`); widgets pull instances with `getIt<EmailBloc>()`. Flow: `ContactPage` (`BlocConsumer<EmailBloc, EmailState>`) → `EmailBloc` → `EmailRepository` → `EmailApi`. `EmailApiImpl.sendEmail` does two things: writes the message to the Firestore `contact` collection **and** sends mail through EmailJS (service `sendgrid`, template `contact_me`). The repository returns `Either<EmailFailure, Email>` (dartz).

### Code generation
`*.freezed.dart` files (bloc state/event, `email_failure`, email model) are generated by `build_runner` from `@freezed` annotations — never edit them by hand; change the source and re-run build_runner.

### Responsive layout
The site adapts between mobile and desktop. Use the helpers in `core/layout/adaptive.dart` (`isDisplayDesktop`, `isDisplayMobile`, `isDisplayMobileOrTablet`, `isDisplaySmallDesktop`, …) rather than checking `MediaQuery` sizes inline; they encode the project's breakpoints and combine the `layout` and `responsive_builder` packages.

## Deployment

GitHub Actions, all manually triggered (`workflow_dispatch`; push triggers are commented out). `main.yml` fans out to `deploy-firebase.yml` (builds web + `firebase deploy --only hosting`) and `deploy-github-pages.yml` (builds web, rewrites `<base href>` to `/My_Portfolio/`, publishes `build/web`). Both recreate the `dotenv` file and pass `--dart-define`s from GitHub secrets. Firebase project id is `m074med`; hosting serves `build/web` with a SPA rewrite to `/index.html` (see `firebase.json`).
